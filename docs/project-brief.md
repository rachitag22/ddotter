# Project Brief

## Product

DDOT Advocacy Map is a public engagement tool for DC transportation projects.

## Audience

- DC transportation advocates
- ANC commissioners and neighborhood leaders
- Residents who want to understand nearby projects
- DDOT staff reviewing public sentiment and project feedback

## Core Hypothesis

A unified, accessible map lowers the barrier for community members to engage with DDOT projects and gives DDOT a clearer signal of public support.

## Problem

DDOT project information is spread across public datasets, official pages, meeting materials, and neighborhood conversations. Even motivated residents may not know which projects are active, where they are located, what stage they are in, or how to submit useful feedback.

## Solution

Create one public web app with:

- A searchable project map
- A filterable project list
- Detail pages for each project
- Lightweight feedback collection
- Public aggregate support signals
- A backend sync process that keeps project data current

## Success Signals

- Advocates can find projects by ward, type, status, and search.
- A user can submit feedback on a project in under one minute.
- Each project shows simple social proof such as support count or comment count.
- DDOT-facing exports or dashboards can summarize feedback by project.
- ArcGIS downtime does not break the public app.

## Non-Goals For MVP

- Full DDOT staff workflow management
- User accounts
- Moderation queue
- Email digests with custom scheduling
- Guaranteed completeness for the art installation layer while endpoint is TBD
- Editing official project records from the app
